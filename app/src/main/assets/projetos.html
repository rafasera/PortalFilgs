Padronização de LOGS e APIs
Fev. de 2020 - Jun de 2020
Descrição do Projeto
Criar um padão para APIs E LOGs de Aplicação
Neste projeto de padronização, foi discutido com os times de desenvolvimento, modelos padronizados de APIs para aplicações, seguindo boas praticas de mercado e uma estrutura unica a fim de simplificar a integração com outras soluções.
Foi definido um padrão para erros bem estruturado com rastreabilidade e melhor definição do evento (Seguindo recomendações do BACEN).
Os LOGS foram estruturados para além do uso de UUID, possuírem em seu conteúdo informações de geolocalização, e uma lista dos eventos segmentada por CDN, Barramento e Aplicação.
Como resultado conseguimos obter um alto grau de rastreabilidade e ainda servir informações para o time de DataScience para analises de previsibilidade.

Fev de 2019 – Mar de 2019
Descrição do Projeto:
Construir Aplicativo Android voltado a apresentar o conteúdo do site filgs.com.br.
IDEIA: Conforme analises obtidas a partir do Google Analytics, foi identificado que 82% do publico que acessa o site tem origem a partir de dispositivos moveis, destes 71% Android e 27% IOS e 2% Outros. Projeto para criação do modelo de monitoração da Experiência de Usuário
mar de 2018 – Jan de 2019
Descrição do projeto:
Projeto consistiu em realizar a criação de painéis que medissem a experiência do usuário enquanto utilizam a solução.
IDEIA: Percebendo que os modelos usuais para medir o desempenho (simulação), apesar de serem efetivos para o processo de monitoramento da aplicação, não conseguiam entregar uma análise real do tempo gasto por cada usuário durante o round-trip da aplicação (início da função | termino da função), para solucionar este ponto criamos gatilhos dentro das páginas a qual o cliente percorre, nestas fazemos duas marcações uma inicial e outra final, em seguida salvamos este dado em uma base relacional que posteriormente será usada para levantar os dados conforme as regras de negócio para cada serviço /produto.
Com a captura das informações é realizada a análise das mesmas e definidos alarmes de “desempenho” e análises de “outlines” para entender horários de picos e origem das maiores requisições.
No step seguinte utilizamos a ferramenta Zabbix para capturar as informações tratadas do Banco de Dados e assim criar alarmes para acionamento junto ao time de eventos.
Como último step foram criados painéis para visualização da diretoria da empresa e tomada das decisões estratégicas com base nas métricas e dados obtidos.
Projeto continuo (o mesmo é replicado para todo novo negócio ou produto desenvolvido).

Monitoramento Estratégico para tomada de decisão.
nov de 2018 – dez de 2018
Descrição do projeto:
Projeto consistiu em realizar a criação de painéis para tomada de decisões para os 5 (cinco), principais clientes da Empresa Linx-SA.
Para esse foram utilizadas as ferramentas Zabbix, Prometheus e Azure Log Analytics.
A partir do levantamento dos dados obtidos pelas soluções citadas, conseguimos obter as métricas de infra, e desempenho e logo em seguida foi utilizado um algoritmo para localizar “Outline” nos processos e realizar as devidas tratativas antes que os mesmos se tornassem incidentes.
Como último step foram criadas duas visões uma técnica para uso dos times especializados e outra Gerencial a qual apresenta o status e andamento dos Outlines.
Para o desenvolvimento das camadas de apresentação foi usada a ferramenta Grafana.

Medir Análise de Desempenho de Métricas de Monitoramento.
set de 2018 – nov de 2018
Descrição do projeto:
Projeto consistiu em realizar a captura de informações armazenadas na ferramenta de monitoramento Zabbix.
Modelo inicial usado neste projeto disponível em meu site link.
Modelo atualmente usado utiliza o serviço Azure - DataBrics, com um formato 5x mais performático e que salva as informações em uma base SQLServer Azure (PaaS).
Toda a codificação deste foi realizada com a linguagem Python.
Com os dados capturados aplicamos a análise de desempenho conforme cada produto, e com isso conseguimos mapear quantas amostras ficaram dentro dos quadros de “ótimo”, “bom” ou “ruim” e o horário em que houve o outline.
Com essa informação é gerado um relatório para uso da equipe de incidentes realizar o mapeamento de desempenho junto aos P&Ds de cada solução para aplicar as devidas melhorias quando cabíveis.

Aplicativo Álcool ou Gasolina
mar de 2018
Descrição do projeto:
Projeto consistiu em criar um aplicativo de fácil entendimento para saber qual a melhor opção para abastecer o seu carro Flex com base nos valores inseridos, seguindo a seguinte lógica:
A principal diferença de preços e vantagens entre os dois combustíveis está na proporção preço X desempenho.
Por exemplo, um carro que faz 10 km/litro com gasolina, costuma fazer em média, 7 km/litro se abastecido com álcool.
Portanto, para o álcool ser mais vantajoso do que a gasolina, o preço do litro tem que custar até 70% do litro da gasolina.
